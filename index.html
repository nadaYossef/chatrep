e& SR Assistant

Welcome

This portal helps extract business rules and support the detection and prediction of SR (service request) anomalies for e&. Use the navigation to manage rules and documentation. Over time this site will help build a chatbot and an automated warning system.

Public & safety notice

  • This site is public. It intentionally does NOT include any credentials, direct database connections, or automated actions.
  • Purpose: (1) implement and share business rules, (2) make onboarding easier for turnover staff, and (3) collect structured feedback and data to design a future chatbot and warning system.
  • Data entered here is for demonstration and collaborative review. Peers can correct or add rules if they see new SR patterns.

About the chatbot & warning system (how this will evolve)

This site is the foundation for a multi-stage project:

  1. Collect business rules and structured SR records from users.
  2. Use those records to build a simple chatbot prototype that accepts an input (for example, a dial/phone number or SR id) and returns a diagnosed issue with reasoning and evidence taken from the business rules and stored data.
  3. Collect user feedback and star ratings to measure accuracy.
  4. Train supervised NLP models (and fine-tune an LLM) on the labeled examples and accuracy annotations to produce a human-like assistant that recommends steps.
  5. When a scenario is repeatedly highly rated, we may offer automation; otherwise continue to provide recommended steps and ask users to verify before action.

Chatbot simulation (read-only)

The first chatbot prototype will be read-only and simulate interpretation. Example: 

Entered dial or number: 0123456789
Diagnosed issue: Packet loss detected on last-hop
Reasoning / evidence: Repeated timeouts in last 3 probes; high retry count in SR row 'retries'; matches rule 'last-hop packet loss'.

Please enter your feedback below.
Accuracy (1-5): [ ]
If incorrect, please write the correct detected issue and the correct steps taken.

Notes: the next interactive version will accept live inputs, call the trained model, and speak recommendations. Training uses exported examples and accuracy ratings (supervised learning + LLM fine-tuning).

Project Workflow (embedded)

The following embeds the Project 1 workflow as structured text with icons and enhanced readability. This is the same written workflow but presented as interactive content.

Phase 1 — Gather & Structure Knowledge
  • 🔎 Input: existing business knowledge, first-line tools, open-only tickets, previously resolved tickets, team documentation.
  • 🛠 Process: collect business rules, export business-table, gather documentation (PDFs, slides).
  • 📤 Output: business knowledge table, documentation tab, examples (issue records).
Phase 2 — Implement & Test System
  • ⚙️ Input: business-table, exported queue & SQL queries.
  • 🧪 Process: implement firstline tool to apply rules, generate explanations and recommended actions, collect feedback (feed.csv).
  • 📤 Output: system outputs, explanation snippets, feedback CSV for study.
Phase 3 — Study Data & Train ML
  • 📈 Input: feed.csv, business-table, dynamic SR DB.
  • 🧠 Process: train supervised NLP & model on labeled feedback to improve accuracy.
  • 🚀 Output: trained model, improved chatbot that can recommend and (optionally) automate tasks with user approval.
